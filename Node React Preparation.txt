useState() — Handling Data & UI Changes
=====================================
React is state-driven — your UI updates when data (state) changes.
The useState() hook is used to store and update data inside a component.


import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0); // initial value = 0

  const increment = () => {
    setCount(count + 1); // updates count → triggers re-render
  };

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={increment}>Increment</button>
    </div>
  );
}

export default Counter;


Part	Meaning
useState(0) - 	Create a state variable count with initial value 0
setCount()	- Function to update the state
count + 1	- New state value
When you call setCount, React re-renders the component with updated state	


Every time setCount() is called → React:
---------------------------------------
Updates the internal state.
Re-renders the component.
Reflects new data in the DOM automatically.

=======================================
React components can:

Fetch data,

Dependency array	When it runs

Example Combining All:
=====================

import { useState, useEffect } from 'react';

function App() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch('http://localhost:5000/api/users')
      .then(res => res.json())
      .then(data => setUsers(data));
  }, []); // runs once when component loads

  return (
    <div>
      <h1>User List</h1>
      {users.map(u => (
        <p key={u.id}>{u.name} - {u.email}</p>
      ))}
    </div>
  );
}

export default App;

11.Context API ?
The Context API in React is a way to share data across components without having to pass props manually at every level (prop drilling).
No need to pass props manually through intermediate components.

Redux:
Redux is a full-featured state management library for global, complex state, while Context is a simpler built-in React API for passing state through the component tree.
✅ Redux pros: Great for large apps with multiple components reading/updating state.
Summary: Use Redux for large, complex apps; use Context for simple state sharing.

memo:

React.memo “memoizes” a component so that React reuses the last rendered output if props haven’t changed, reducing unnecessary rendering and improving performance.
